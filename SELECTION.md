# 选型说明 / Trade-off

## 背景与目标
为工业园区做语音访客登记，替代保安人工问询。来电者电话接入，agent 在自然对话中采集 车牌号 / 来访单位 / 手机号 / 来访事由，把结构化登记推送到保安端，全程 25 秒内，体验要像真人门卫。系统面向中国部署、开源。本项目的考察重点是技术选型与 trade-off 判断，下面逐项说明每个选择以及它的代价。

## 选型总览

| 层 | 选择 | 一句话理由 |
|---|---|---|
| 编排 | Pipecat（自托管） | 中国可托管、内置打断、可插桩 harness、开源 |
| STT | 百炼 Paraformer 实时 | 国内直连低延迟，与 LLM/TTS 同一把 key |
| LLM | Qwen（qwen-plus/turbo） | 中文强、国内直连、走 OpenAI 兼容端点免封装 |
| TTS | 百炼 CosyVoice（flash） | 流式低延迟，同一把 key |
| 音频入口 | 本地音频 + 虚拟声卡 | 解耦下游，一套管线吃多种来电源 |
| 数据库 | 本地 PostgreSQL | 零注册零跨境，clone 即复现 |
| 推送 | 企业微信群机器人 webhook | 官方、免费、合规 |

## 关键 trade-off

### 编排：Pipecat，而非 VAPI/Retell 或纯手搓
VAPI、Retell 这类商业 SaaS 是美国云托管、无法自托管。人在中国用它等于把实时音频跨境往返美国云，而且它们的电话底层是 Twilio，中国不通；项目要开源，闭源 SaaS 也开不了。另一个极端是纯手搓 STT 加 TTS，这会漏掉语音 agent 最难的一块：轮次检测与打断，容易做成对讲机式的轮流卡顿，直接违反题目的对话体验标准。Pipecat 自托管、能跑在中国机器、内置 Silero VAD 和中断处理、各服务可插拔，而且 harness 能织进 STT 到 LLM 到 TTS 之间。代价是比一键 SaaS 多花搭建和调试时间，在三天窗口内可接受。

### 全栈国产：阿里云百炼一把 key
实时音频经不起代理带来的延迟和抖动，住宅代理只对文本和控制面有用，救不了逐帧的语音流。所以 STT、TTS、LLM 全部收在阿里云百炼，国内直连。百炼自带 Paraformer 实时识别、CosyVoice 流式合成和 Qwen，三层共用同一把 DASHSCOPE_API_KEY，省掉了智能语音交互 NLS 那套 AccessKey 加 AppKey 加 token 的鉴权流程。代价是 Pipecat 没有现成的阿里语音 service，STT 和 TTS 各需自写一层薄封装；LLM 走 OpenAI 兼容端点，可直接用框架自带 service，免封装。

### 电话：不逃课，用 Continuity 跑通真实来电
题目把电话方案当考点，明说不能绕。现实是个人拿不到可编程的 +86 入呼号码（需要企业资质），Twilio 在中国不通，Google Fi 这类消费级手机卡也做不到程序自动应答。MVP 阶段用 iPhone Continuity 把打到 iPhone 那张卡的真实来电甩到 Mac 接听，音频经虚拟声卡进管线，零电信资质、零月租地跑通了一通真电话，正面回应了"用你自己的电话拨打、Agent 接听"。中国侧测试再用微信语音接入，免费、人人都有、国内可达。已知局限是 Continuity 本身不够稳定、要两台苹果设备同网靠近、演示时手动接听。生产形态会换成正经电信 SDK，中国就是阿里云语音加企业资质拿正式号码。

### 微信推送：企业微信群机器人 webhook
个人微信自动化（QClaw、OpenClaw 一类）是灰产路线，封号、不稳、不合规，主动弃用。企业微信群机器人 webhook 是官方能力，免费，一个 HTTP POST 就把结构化消息发进群，合规且可靠。演示里这个群就代表保安端。

### 数据库：本地 Postgres，而非 Supabase
开源加公司在中国测试这两个前提下，本地 docker Postgres 意味着别人 clone 下来 `docker compose up` 就有库，零注册、零跨境、可复现。Supabase 的托管 auth 和即时 API 在本项目用不上，反而给中国测试者多加一个境外依赖和注册步骤。

### 对话设计：批量抽取加复述确认，而非逐字段问答
题目的对话体验标准明确反对机械的一问一答。做法是用一个系统提示让 LLM 从一句话里抽出多个字段、只追问缺失的、结尾复述确认。车牌和手机号的 ASR 错误率偏高（实测把"蓝色鲸鱼"识别成"蓝色金鱼"），所以关键字段复述确认是必需的设计而非可选项。

## 已知局限与生产演进
- 电话：Continuity 是 MVP hack，生产用阿里云语音加企业资质拿正式 +86 入呼号。
- 部署：语音入口依赖真实机器的声卡和 GUI，无头云 VM 跑不了完整语音 demo；生产侧电话改走 SDK 之后这条限制消失。
- 回访识别、门卫查询 agent、多路并发列为加分项，核心闭环稳定后再加；回访沿用同一张登记表，无需改 schema。
